The configuration module¶

Tools for keeping track of system settings.

The configuration module provides a system for managing the general, reusable settings for the software—both a collection of default settings and some user-specific settings—including default file locations, graph colors, graph update delays, and so on. It keeps track of these data in memory and saves them to (and initially reads them from) a file on the disk.

The module defines some classes for the above-mentioned purposes, and it creates an actual instance of the Configuration class, so that other modules can access the configuration without having to create a new instance and, therefore without having to re-read the file from the hard drive.

The Configuration class is a container for system-wide settings concerning appearance of certain components, default file locations, and helpful ways for letting users know that an experiment is finished or that serious system errors (e.g. magnet quenches) have occurred.
